Nonlinear Feature Map

To creat a nonlinear method for regression or classifier is to transform feature vector via a nonlinear feature map

Φ:Rd→Rm(1.1)\Phi: \mathbb{R}^d \rightarrow \mathbb{R}^{m} \tag{1.1} Φ:Rd→Rm(1.1)

This way is pretty intuitive since we can just image adding some features on the original feature vector. Then apply this feature map over whole dataset get (Φ(xi),yi)∀i)(\Phi(x_{i}),y_{i})\quad \forall i)(Φ(xi​),yi​)∀i)

Thus, in regression, the classifier is :

f(x)=wTϕ(x)+b(1.2)f(\boldsymbol{x})=\boldsymbol{w}^{T} \boldsymbol{\phi}(\boldsymbol{x})+b \tag{1.2} f(x)=wTϕ(x)+b(1.2)

Where w∈Rmb∈R\boldsymbol{w}\in \mathbb{R}^m\quad b \in\mathbb{R}w∈Rmb∈R

Inner Product Kernels

  • Many ML algorithms depend on ϕ(x)\boldsymbol{\phi}(\boldsymbol{x})ϕ(x) only via inner products

    ⟨Φ(x),ϕ(x′)⟩(1.3)\langle \Phi(x),\phi(x^{\prime})\rangle \tag{1.3} ⟨Φ(x),ϕ(x′)⟩(1.3)

  • for certain Φ\PhiΦ, the function

    k(u,v)=⟨Φ(u),Φ(v)⟩(1.4)k(u,v)=\langle\Phi(u),\Phi(v)\rangle \tag{1.4} k(u,v)=⟨Φ(u),Φ(v)⟩(1.4)

    can be computed efficiently even m is huge or possibly infinite. k is called an inner product kernel.

Some important kernels

Homogeneous polynomia kernel

k(u,v)=(uTv)p(1.5)k(u,v)=(u^Tv)^{p} \tag{1.5} k(u,v)=(uTv)p(1.5)

Inhomogeneous polynomia kernel

k(u,v)=(uTv+b)p(1.6)k(u,v)=(u^Tv+b)^{p} \tag{1.6} k(u,v)=(uTv+b)p(1.6)

Gaussian kernel (not list here)

SPD kernels

  • One way to determine an inner product kernel is to construct Φ\PhiΦ explicitly.

  • Another way is to verify that k is an kernel if it satisfies the following properties

    • k:Rd×Rd→Rk:\mathbb{R}^{d}\times \mathbb{R}^{d} \rightarrow\mathbb{R}k:Rd×Rd→R. k is symmetric i.e. k(u,v)=k(v,u)∀u,vk(u,v)=k(v,u) \quad\forall u,vk(u,v)=k(v,u)∀u,v

    • kkk is positive definite if:

      [k(x1,x1)⋯k(x1,xn)⋮⋱⋮k(xn,x1)⋯k(xn,xn)]\left[\begin{array}{ccc} k\left(\boldsymbol{x}_{1}, \boldsymbol{x}_{1}\right) & \cdots & k\left(\boldsymbol{x}_{1}, \boldsymbol{x}_{n}\right) \\ \vdots & \ddots & \vdots \\ k\left(\boldsymbol{x}_{n}, \boldsymbol{x}_{1}\right) & \cdots & k\left(\boldsymbol{x}_{n}, \boldsymbol{x}_{n}\right) \end{array}\right] ⎣⎢⎡​k(x1​,x1​)⋮k(xn​,x1​)​⋯⋱⋯​k(x1​,xn​)⋮k(xn​,xn​)​⎦⎥⎤​

      is a PSD matrix for all n∈Nn\in \mathbb{N}n∈N, and x1...xn∈Rdx_{1}...x_{n}\in\mathbb{R}^{d}x1​...xn​∈Rd

  • Theorem: k is an SPD kernel iff k is an inner product kernel.

The Kernel Trick

A machine learning algorithm is said to be kernelizable if it is possible to formulate the algorithm such that all training instances xix_{i}xi​ and any test instance xxx occur in inner products of the form⟨xi,xj⟩,⟨xi,x⟩,or⟨x,x⟩\langle x_{i},x{_{j}} \rangle,\langle x_{i},x \rangle,\text{or}\langle x,x \rangle⟨xi​,xj​⟩,⟨xi​,x⟩,or⟨x,x⟩

Since we can suppose Φ\PhiΦ is a feature map associatecd to an inner product kernel k, if we apply a kernelizable algorithm to the training data like constructing :

(Φ(x1),y1),…,(Φ(xn),yn)(1.7)\left(\boldsymbol{\Phi}\left(\boldsymbol{x}_{1}\right), y_{1}\right), \ldots,\left(\boldsymbol{\Phi}\left(\boldsymbol{x}_{n}\right), y_{n}\right) \tag{1.7} (Φ(x1​),y1​),…,(Φ(xn​),yn​)(1.7)

then we can formulate the algorithm such that transformed feature vectors only appear via inner products ⟨Φ(x),Φ(x′)⟩\langle\boldsymbol{\Phi(x),\Phi(x^{\prime})}\rangle⟨Φ(x),Φ(x′)⟩ with other similar transformed feature vectores.

  • this can be implemented by evaluating k(u,v)=⟨Φ(u),Φ(v)⟩k(u,v)=\langle\Phi(u),\Phi(v)\ranglek(u,v)=⟨Φ(u),Φ(v)⟩ which eliminates the need to ever compute Φ(x)\Phi(x)Φ(x).

Kernel Ridge Regression

Ridge regression is kernelizable. use the kernel trick to extend it to a nonlinear method called ridge regression.

Kernel ridge regresion(without offset)

since some Φ\PhiΦ contain a constant term, as with the inhomogeous polynomial kernel, the offset is not always needed

For the KRR(no offset), we have objective function

min⁡w1n∑i=1n(yi−wTxi)2+λ∥w∥2=1n∥y−Xw∥2+λ∥w∥2∝wT(XTX+nλI)w−2yTXw+yTy(1.8)\begin{aligned} \min _{w}\frac{1}{n} \sum_{i=1}^{n}\left(y_{i}-\boldsymbol{w}^{T} \boldsymbol{x}_{i}\right)^{2}+\lambda\|\boldsymbol{w}\|^{2} &=\frac{1}{n}\|\boldsymbol{y}-\boldsymbol{X} \boldsymbol{w}\|^{2}+\lambda\|\boldsymbol{w}\|^{2} \\ & \propto \boldsymbol{w}^{T}\left(\boldsymbol{X}^{T} \boldsymbol{X}+n \lambda \boldsymbol{I}\right) \boldsymbol{w}-2 \boldsymbol{y}^{T} \boldsymbol{X} \boldsymbol{w}+\boldsymbol{y}^{T} \boldsymbol{y} \end{aligned} \tag{1.8} wmin​n1​i=1∑n​(yi​−wTxi​)2+λ∥w∥2​=n1​∥y−Xw∥2+λ∥w∥2∝wT(XTX+nλI)w−2yTXw+yTy​(1.8)

The solution is

w^=(XTX+nλI)−1XTy(1.9)\widehat{\boldsymbol{w}}=\left(\boldsymbol{X}^{T} \boldsymbol{X}+n \lambda \boldsymbol{I}\right)^{-1} \boldsymbol{X}^{T} \boldsymbol{y}\tag{1.9} w=(XTX+nλI)−1XTy(1.9)

Where

y=[y1⋮yn],X=[x1T⋮xnT]∈Rn×d\boldsymbol{y}=\left[\begin{array}{c} y_{1} \\ \vdots \\ y_{n} \end{array}\right], \quad \boldsymbol{X}=\left[\begin{array}{c} \boldsymbol{x}_{1}^{T} \\ \vdots \\ \boldsymbol{x}_{n}^{T} \end{array}\right] \in \mathbb{R}^{n \times d} y=⎣⎢⎡​y1​⋮yn​​⎦⎥⎤​,X=⎣⎢⎡​x1T​⋮xnT​​⎦⎥⎤​∈Rn×d

Note XTXX^TXXTX is not the gram matrix of the training data (it belongs to Rd×d\mathbb{R^{d\times d}}Rd×d), thus we need to further transform the solution

  • apply matrix inversion lemma:

    (P+QRS)−1=P−1−P−1Q(R−1+SP−1Q)−1SP−1(1.10)(\mathbf{P}+\mathbf{Q R S})^{-1}=\mathbf{P}^{-1}-\mathbf{P}^{-1} \mathbf{Q}\left(\mathbf{R}^{-1}+\mathbf{S P}^{-1} \mathbf{Q}\right)^{-1} \mathbf{S P}^{-1} \tag{1.10} (P+QRS)−1=P−1−P−1Q(R−1+SP−1Q)−1SP−1(1.10)

After simplification we get

w^T=yT(μI+XXT)−1X(1.11)\widehat{\boldsymbol{w}}^{T}= \boldsymbol{y}^{T}\left(\mu \boldsymbol{I}+\boldsymbol{X} \boldsymbol{X}^{T}\right)^{-1} \boldsymbol{X} \tag{1.11} wT=yT(μI+XXT)−1X(1.11)

Note the Gram matrix G=XXTG=XX^{T}G=XXT appears, although the method is still not kernelized because of the matrix XXX . However, this can be resolved by taking the inner product of w^\widehat{w}w with a test instance xxx . Thus introduce the notation

G:=[⟨x1,x1⟩⋯⟨x1,xn⟩⋮⋱⋮⟨xn,x1⟩⋯⟨xn,xn⟩]g(x):=[⟨x1,x⟩⋮⟨xn,x⟩]\boldsymbol{G}:=\left[\begin{array}{ccc} \left\langle\boldsymbol{x}_{1}, \boldsymbol{x}_{1}\right\rangle & \cdots & \left\langle\boldsymbol{x}_{1}, \boldsymbol{x}_{n}\right\rangle \\ \vdots & \ddots & \vdots \\ \left\langle\boldsymbol{x}_{n}, \boldsymbol{x}_{1}\right\rangle & \cdots & \left\langle\boldsymbol{x}_{n}, \boldsymbol{x}_{n}\right\rangle \end{array}\right] \quad \boldsymbol{g}(\boldsymbol{x}):=\left[\begin{array}{c} \left\langle\boldsymbol{x}_{1}, \boldsymbol{x}\right\rangle \\ \vdots \\ \left\langle\boldsymbol{x}_{n}, \boldsymbol{x}\right\rangle \end{array}\right] G:=⎣⎢⎡​⟨x1​,x1​⟩⋮⟨xn​,x1​⟩​⋯⋱⋯​⟨x1​,xn​⟩⋮⟨xn​,xn​⟩​⎦⎥⎤​g(x):=⎣⎢⎡​⟨x1​,x⟩⋮⟨xn​,x⟩​⎦⎥⎤​

Then we can have classifier as

f^(x)=w^Tx=yT(XXT+nλI)−1Xx=yT(G+nλI)−1g(x)(1.12)\begin{aligned} \widehat{f}(\boldsymbol{x}) &=\widehat{\boldsymbol{w}}^{T} \boldsymbol{x} \\ &=\boldsymbol{y}^{T}\left(\boldsymbol{X} \boldsymbol{X}^{T}+n \lambda \boldsymbol{I}\right)^{-1} \boldsymbol{X} \boldsymbol{x} \\ &=\boldsymbol{y}^{T}(\boldsymbol{G}+n \lambda \boldsymbol{I})^{-1} \boldsymbol{g}(\boldsymbol{x}) \end{aligned} \tag{1.12} f​(x)​=wTx=yT(XXT+nλI)−1Xx=yT(G+nλI)−1g(x)​(1.12)

This shows that KRR w/o is kernelizable. So we apply kernek trick by simply selecting a kerenel k and replace ⟨u,v⟩\langle u,v\rangle⟨u,v⟩ with k(u,v)k(u,v)k(u,v) . After substituion, GGG and g(x)g(x)g(x) are replaced by

K:=[k(x1,x1)⋯k(x1,xn)⋮⋱⋮k(xn,x1)⋯k(xn,xn)],k(x):=[k(x1,x)⋮k(xn,x)]\boldsymbol{K}:=\left[\begin{array}{ccc} k\left(\boldsymbol{x}_{1}, \boldsymbol{x}_{1}\right) & \cdots & k\left(\boldsymbol{x}_{1}, \boldsymbol{x}_{n}\right) \\ \vdots & \ddots & \vdots \\ k\left(\boldsymbol{x}_{n}, \boldsymbol{x}_{1}\right) & \cdots & k\left(\boldsymbol{x}_{n}, \boldsymbol{x}_{n}\right) \end{array}\right], \quad \boldsymbol{k}(\boldsymbol{x}):=\left[\begin{array}{c} k\left(\boldsymbol{x}_{1}, \boldsymbol{x}\right) \\ \vdots \\ k\left(\boldsymbol{x}_{n}, \boldsymbol{x}\right) \end{array}\right] K:=⎣⎢⎡​k(x1​,x1​)⋮k(xn​,x1​)​⋯⋱⋯​k(x1​,xn​)⋮k(xn​,xn​)​⎦⎥⎤​,k(x):=⎣⎢⎡​k(x1​,x)⋮k(xn​,x)​⎦⎥⎤​

KKK is called the kernel matrix. Now, the final form of the KRR(w/o) offset predictor is

f^(x)=yT(K+nλI)−1k(x)(1.13)\widehat{f}(\boldsymbol{x})=\boldsymbol{y}^{T}(\boldsymbol{K}+n \lambda \boldsymbol{I})^{-1} \boldsymbol{k}(\boldsymbol{x}) \tag{1.13} f​(x)=yT(K+nλI)−1k(x)(1.13)

Note entire process is equivalent to first applying the feature map Φ\PhiΦ to all feature vectors, and applying ridge regression w/o in the new feature space. Because all we need is to obtain a nonlinear regression predictor and once we have selected a kernel all the calculation needed for predicting a new point is implicitly finished.

Computational Complexity of KRR w/o

The computational complexity of KRR without offset is O(n3)O(n^3)O(n3) which comes from having to invert an n×nn\times nn×n matrix. As with regular regression, this can be accelerated using gradient descent and related methods
