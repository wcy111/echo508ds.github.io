三、创建虚拟环境

-虚拟环境简单介绍

目前我理解的创建虚拟环境的动机主要是 1.不同项目或者库的依赖不同，同一个包在不同项目中对应版本的不同，因此很难兼容，频繁的去upgrade或者downgrade明显过于繁琐。 2. 一个团队大家共用一个账号，如果共用一个环境显然会变得更加混乱。因此创建虚拟环境就十分有必要，使用时激活，环境配置都在激活的环境中进行，环境之间互不影响，并且可以本地或者远程clone别人的环境来进行自己的部署开发。

事实上，环境管理的工具很多，有virtualenv, Pipenv, conda,docker等。我们熟知的pip是包管理工具, virtualenv可以管理环境。而conda两者兼有，因此这里我们使用conda,conda还有miniconda，其中后者更加轻便。

-常用命令

conda list

类似于pip list,可以查看当前环境目录安装的所有库

conda info -e

查看创建的所有环境，会显示所有环境的名字

conda activate env_name

激活环境, env_name是环境的名字

conda deactivate env_name

关闭环境

四、本地连接服务器端Jupyter notebook

这里简述实现流程，具体内容也可参考官方文档[3]

- 在服务器端配置jupyter configuration

**第一步,**生成jupyter笔记本的默认配置文件，如果已经存在会提示你是否要初始化。

**第二步，**设置笔记本的登录密码(免密登录的设置参考官方文档)

**第三步，**修改jupyter笔记本配置文件，用文本编辑器打开jupyter_notebook_config.py文件，将其中对应行注释符#去掉修改字段或者直接在文件任一位置添加:

这里文本编辑器使用vim.关于vim的操作可以参考4

- 在服务器端开启jupyter server

完成基本设置好，然后在中bash输入

jupyter notebook

默认的端口为8888，但是有时候已经有人使用了该端口，为了避免冲突可以指定开启端口号

如果在本机这样操作会直接在浏览器跳出jupyter web页面，在服务器端因为没有GUI并且在config中设置了 no browser，因此 bash 会提示server 运行在B端口。接下来我们就可以放着先不管了。

-创建本地到服务器端的映射

事实上也可以改jupyter config的一些配置，直接远程访问该jupyter（remote address:YYYY），这里由于某些原因我失败了暂时选择了如下方法。

在本地终端输入:

ssh -N -f -L localhost:XXXX:localhost:YYYY team6@192.168.156.31

-N 告诉SSH没有命令要被远程执行

-f 告诉SSH在后台执行

-L 指定port forwarding的配置 远程端口是YYYY 本地是XXXX

相当于一个映射，把服务器地址映射到 localhost的一个端口。

随后打开任意浏览器地址栏输入localhost:XXXX并输入密码 xxxx（这是我之前在jupyter config里设置的）,即可进入 。

这里如果我们想在jupyter上运行Shell命令，只需要在代码运行的cell前面加! , 如果需要交互 可以在末尾加–yes 或者-- yes *

五、上传下载文件

如果是使用了远程终端软件，图形界面内自备了可视化接口就可方便上传下载文件。在bash中我们可以使用shell命令scp进行操作。
